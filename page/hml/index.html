HML - Hardware Middle Layer

Page update at 2018-03-28

What is HML?

HML is a lite and easy-use firmware library & BSP set for some MCUs and boards,such as 8051 and STC89 series MCUs now(refer to support-list for details).It coded with C language and compiled by SDCC compiler.HML aimed to provide a bridge from originality to hardware,help people develop more quickly.Besides,we found there is lack of application examples about SDCC on the Internet,even if SDCC is free and high-efficiency.So we decide to code HML to exploit the field and share our experience.HML opens all source files on GitHub.Welcome advice from you!

What MCU or board is being supported by HML now?

The lastest version code of HML is v0.0.0,and the followings is support list of current version and related component in HML:

  • MCUcorresponding component
  • Intel MCS-51 based microprocessors (8031, 8032, 8051, 8052, etc.)
  • STC micro STC89 series (STC89C52RC, STC89C54RD, etc.)
  • BOARD corresponding component
  • ...

How to start?

STEP-1 requierd-environment and tools

You need some prepartions about configuring tools before using HML.

Firstly,you need a computer with fitted operation system.HML is known to compile from the source code on the following platforms:

  • Linux - x86_64
  • Microsoft Windows - x86
  • Microsoft Windows - x86_64
  • Rasbian (Debian for Raspberry Pi) - ARMv6

In fact,it may be Ok on other platforms as long as it can run SDCC.However,we don't have such conditions to verify.If run HML successfully on the other platforms,you can tell us via mcu@zhishan-iot.ga.

Secondly,install the lastest version of SDCC according to your OS and refer to support-list to download component of HML which is fitted with your MCU or board.In order to make sure that SDCC has been installed rightly,users of Linux need to enter sdcc --version in terminal;and users of Windows need enter the same code in cmd.exe.Then if it shows version information of SDCC like following screenshot,it means all is Ok!


Note for users of Windows

Users may need to configure environment variable of SDCC manually!

STEP-2 get start with HML

In the beginning of this step,you need to know the file structment of HML.Every HML component has similar structment,there may be few subtle distinctions amony them.Now we take HML_FwLib_8051 for example.

File structment of HML_FwLib_8051

HML_FwLib_8051
├── inc
├── obj
├── sample
├── src
├── user
└── errormap.txt

The tree shows the structment of HML_FwLib_8051,it includes inc,obj,sample,src,test folders and errormap.txt.Details about them show in the following form.
Name Description
inc include all header files(*.h) of HML
obj store all output files,incluing *.hex,*.ihx,*.lk,*.rel, during compiling
sample provide some sample files with *.c format to help users learn about HML
src store all source files(*.c) of HML
user store makefile and a source file which includes main function
errormap.txt record all error-code information defined by HML, they may appear during compiling
#Note:
 *makefile's name are all added platform-name in the end for different platforms,such as makefile_win_x64
 *default name of the source file with main function in user folder is test.c

There are two ways to start your project with HML,they are #QuickStart and #Custom.

QuickStart: user just need modify code in test.c,and use default makefile,then it will comlete all work.After makefile script complete compiling,it will show these information like following image.If you computer support beeper,the script will enable the beeper to notify you all works are completed,and you can disable this function by delete related line in script file.


Custom: if you have enough abilities or want to do more things,you can just add inc and src folders into your project and write your own makefile.
