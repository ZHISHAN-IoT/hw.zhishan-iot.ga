HML - HML_FwLib_8051

A lite firmware library for 8051 MCUs.

Brief

HML_FwLib_8051 is the first member component of HML firmware libraries, providing a group of functions for users to handle the on-chip resources of 8051 MCUs, including GPIO, external interrupt, timer and UART modules. The 8051 MCUs have been one of the most widely used MCUs since it was produced by Intel in 1981. As amateurs with some experience of 8051 development, we intend to develop a lite and easy-use tool that can help future developers to complete projects based on 8051 MCUs more easily and quickly.

The source codes are written in C language and for SDCC compiler, i.e., it can only be compiled by SDCC. This choice is motivated by the fact that SDCC is free and highly efficient, while there are very few examples of application about SDCC on the Internet. We hope that, as one of the early attempts to develop MCU projects using SDCC, this work will make SDCC become more popular among MCU developers.

Feature

The HML is an opensource project which is licensed under WTFPL2.

To the best of our knowledge, it's the first MCU firmware library developed base on SDCC compiler.

More readable code, no need to care about internal registers of 8051 MCU.

How to start?

STEP-1 requierd-environment and tools

You need some preparations before using HML_FwLib_8051.

Firstly, HML_FwLib_8051 needs a proper development environment. The following table lists the verified environments on which the source files of HML_FwLib_8051 can be successfully compile:

  • Linux - x86_64
  • Microsoft Windows - x86
  • Microsoft Windows - x86_64
  • Rasbian (Debian for Raspberry Pi) - ARMv6

Any other environments on which the SDCC can run may also be available. But we are not capable to verify so many environments. So, if you have compiled HML successfully on other environments, you can tell us via mcu@zhishan-iot.ga.

Secondly, install the lastest version of SDCC according to your OS. In order to make sure that SDCC has been installed rightly, users of Linux need to enter sdcc --version in terminal; and users of Windows need enter the same code in cmd.exe. Then if it shows version information of SDCC like following screenshot, it means all is Ok!


Note for users of Windows

Users may need to configure environment variable of SDCC manually!

Besides,we recomand use GNU Make to help generate target file.There is a file named "Makefile" under user directory in advance.

STEP-2 get started with HML_FwLib_8051

At the beginning of this step,you need to know the file structure of HML_FwLib_8051.

File structment of HML_FwLib_8051

HML_FwLib_8051
├── doc
├── inc
├── obj
├── sample
├── src
└── user

The tree shows the structment of HML_FwLib_8051,it includes inc,obj,sample,src and user folders. Details about them show in the following form.
Name Description
doc store releated documents about HML_FwLib_8051
inc include all header files(*.h) of HML
obj store all output files,incluing *.hex,*.ihx,*.lk,*.rel, during compiling
sample provide some sample files with *.c format to help users learn about HML
src store all source files(*.c) of HML
user store makefile and a source file which includes main function
#Note :
 *default name of the source file with main function in user folder is test.c

There are two ways to start your project with HML_FwLib_8051,they are #QuickStart and #Custom.

QuickStart: you just need to modify the content of test.c and then enter the make under user directory as shown in the first line of the following figure, the remain work will be completed automatically by Makefile.
